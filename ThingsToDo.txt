=== Item 4: Audit before/after diff ===
docs/08 section 7 promises "actor, action, resource, and diff"; the schema comment and OpenAPI description already claim it, but no writer records a before value.
Convention: update-type mutations write detail["changes"] = {field: {"from": old, "to": new}} (changed fields only). Set-valued mutations write added_*/removed_* id arrays - added/removed IS the diff; membership from/to dumps are noise. Legacy row shapes stay valid (append-only); the reader tolerates both.
- server/internal/audit/audit.go: add type Change struct { From any `json:"from"`; To any `json:"to"` }.
- server/internal/quiz/service.go:
  - UpdateQuiz (~:307-342): old row already loaded in-tx; emit from/to per changed field instead of new-values-only.
  - UpdateQuestion (~:600-627): load the old question row in-tx (quiz row lock already serializes) and diff type/body/options/correct/points/penalty. Including correct is fine - audit is admin-only.
- server/internal/quiz/lifecycle.go:
  - Publish on republish: changes for starts_at/ends_at/duration_sec (old values on the locked row).
  - Extend / ForceClose: migrate from_ends_at/from_status flat keys to the changes convention.
  - Archive (currently nil detail): changes {status: {from: closed, to: archived}}.
  - SetAssignments (~:605): before/after audiences are already computed at ~:530-543 and discarded; add added_user_ids/removed_user_ids alongside the counts.
- server/internal/authusers/admin.go: users.updated (~:158) diffs old vs new into changes; groups.members_set gets added/removed arrays.
- web/src/admin/AuditPanel.tsx (fetches detail but never renders it): per-row expander; detail.changes renders one "field: from -> to" line per field; any other non-null detail pretty-prints as JSON (legacy tolerance).
- Tests: extend existing audit assertions in the quiz/authusers flow tests for UpdateQuiz, UpdateQuestion, Extend, users.updated to require the changes shape.

=== Item 7: Gateway periodic auth revalidation ===
docs/05 section 3 says the gateway "checks can() once at subscribe and revalidates on token refresh". The gateway cannot observe token refresh (the access JWT has no session id and the refresh cookie is path-scoped to /api/v1/auth), so implement periodic revalidation - strictly stronger - and update the doc wording.
All in server/internal/realtime/gateway.go:
1. Package var revalidateInterval = 60 * time.Second next to heartbeatTimeout (:67) - shrinkable in tests. Dedicated ticker, not the 30 s ping tick, so transport liveness and authz stay independently tunable.
2. Close code statusAuthRevoked websocket.StatusCode = 4003 next to statusSessionReplaced.
3. Testability seam (required - existing tests pass nil auth to NewGateway): field lookupUser func(ctx, id) (authusers.User, error), set from auth.UserByID when auth != nil, plus SetUserLookup (mirrors SetAttemptOwner). Nil lookup skips the user-status half; existing tests pass unmodified.
4. Per-route revalidate closures built in each handler:
   - monitor: fresh user exists + Status == "active" + re-run g.owner + Can(ActionQuizWatchLive, ...) with the FRESH user (role downgrade takes effect).
   - attempt: fresh user active + g.attemptOwner(ctx, attemptID) still equals actor.ID.
   - notify: fresh user exists + active.
5. pump and pumpAttempt gain a revalidate func(ctx) error param (nil = skip); a revalidateInterval ticker in each select; on failure conn.Close(statusAuthRevoked, "authorization revoked") and return, with an info log.
6. Load: one users SELECT + one owner lookup per socket per minute - negligible.
- Tests: new gateway_revalidate_test.go (white-box, reuse mountMonitor/mountAttempt/fakeSubscriber, shrink interval to ~50 ms with cleanup restore): active user stays open across 3+ intervals (prove liveness via an emitted event); lookup flipping to disabled closes with 4003; owner flip closes the monitor socket; attempt and notify sockets close on deactivation.
- Docs: docs/05 section 3 - replace "revalidates on token refresh" with periodic re-check wording (a deactivated user or revoked permission closes the socket within a minute).
