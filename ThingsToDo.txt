Smaller design-doc items with no home yet

- Audit log read side - writes happen everywhere, but the admin-facing GET /audit endpoint is missing.
- Live-quiz teacher controls - extend ends_at, force-close early, and archive (the fifth lifecycle state) have no endpoints; only the automatic scheduler close exists.
- Notifications on assignment changes (in-app/email) - nothing exists.
- Single active session per attempt - listed as "always on" in the design; not implemented (depends on the realtime layer).

In short: the server-authoritative core (identity, authoring, scheduling, attempts, grading) is done and solid,but the platform is not yet usable by a student or admin through the browser. The natural next steps in plan order are finishing Milestone 4 (results release + the web attempt player, which is also the exit criterion's user-visible half), then realtime, guardrails/kick, import, and analytics.
