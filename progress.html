Milestone 0 - Database schema and migrations

milestone 0
2026-07-05 · after 2e7dffd
  • Initial schema landed, transcribed 1:1 from docs/03-data-model.md: all 13 tables (users, groups, group_members, quizzes, questions, quiz_assignments, attempts, attempt_answers, imports, attempt_events, audit_log, quiz_stats, student_stats), 8 enum types, and every index from the indexing checklist.
  • Migrations tooling: goose SQL migrations embedded into the macquiz binary via a new migrate mode, guarded by a Postgres advisory session lock so concurrent migrators at deploy time cannot interleave DDL. pgx joins the dependency tree as the database driver.
  • Append-only enforced in the database: triggers reject any UPDATE or DELETE on audit_log and attempt_events (data safety invariant 5), so the rule holds even for the table owner.
  • Compose gating: a one-shot migrate service runs before app and worker start (service_completed_successfully), so the API never takes traffic on a stale schema.
  • Migration lifecycle test: up from zero → schema spot checks → append-only trigger checks → case-insensitive citext email lookup → down to zero → re-up, running against a real Postgres 16 service container in CI and the Compose database locally.
Verified end-to-end: from clean volumes, docker compose up --build migrates then serves; psql \dt shows the full schema; migration test green.

Milestone 0 - Backend skeleton and dev stack

milestone 0
2026-07-05 · after cfe22fa
  • Go monorepo scaffolded under server/: single macquiz binary with serve (API) and worker (jobs) modes, matching the two-container Compose topology from the deployment plan.
  • Module boundaries laid down as packages with documented responsibilities: authusers, quiz, attempt, analytics, realtime, worker - each doc.go states what the module owns and what it must never do, straight from the architecture doc.
  • HTTP layer: chi router with request ID, real IP, logging, panic recovery, and timeout middleware; GET /healthz liveness endpoint reporting version/commit; graceful shutdown on SIGTERM with a 10 s drain.
  • Docker Compose dev stack: Postgres 16, Redis 7 (AOF on), app, and worker; healthcheck-gated startup; non-default host ports (5433/6380) so it coexists with host-local databases.
  • Distroless production image: multi-stage Dockerfile producing a static ~15 MB binary image shared by app and worker.
  • Contract and CI seeded: api/openapi.yaml (spec-first, oapi-codegen-ready) covering /healthz; GitHub Actions running gofmt, vet, tests, and build; Makefile for the common loops.
Exit criterion met: docker compose up serves the health check; unit tests green.
